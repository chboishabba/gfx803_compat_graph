AssertAIGreaterThanEqual in the next build if it makes
zero difference to the Tensile compiler. This particular
option allows ROCm to choose kernels based on AI
(arithmetic intensity), and i think it's safe to ignore this
one. will confirm on the next build test.
Another thing you can try for specific Linux builds is to
the change the:
WorkGroupMapping: -1
WorkGroupMappingType: Z
to:
WorkGroupMapping: 0
WorkGroupMappingType: A
(zero, and letter A,). As this fix (in theory) is only
required for Windows and adds extra code in the
kernels, so for Linux it maybe faster and better to use
hardware mapping, and default workgroup
distribution.
